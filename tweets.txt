TWEET 5 — Profiles + encryption
-------------------------------------------------------------------
multi-profile system is live.

each profile is:
→ isolated session storage
→ UUID folder name on disk (not your name, not a timestamp)
→ AES-256-GCM encrypted prefs
→ scrypt-derived keys (now upgraded to Argon2id — 64MB memory cost)

switching profiles doesn't bleed data between them.

TWEET 6 — The panic button
-------------------------------------------------------------------
the panic button.

double-click → instant wipe. no animation. no confirmation. no toast.

what it does in under 1 second:
→ clears all storage (cookies, cache, indexedDB, everything)
→ destroys every WebContentsView
→ clears clipboard
→ kills all messaging rooms
→ resets the UI to a clean dashboard

also works via:
→ Cmd+Shift+X keyboard shortcut
→ trackpad shake (5 direction changes in 600ms)
→ remote trigger from your phone via local network message

you should never feel trapped with sensitive data on screen.

TWEET 7 — Decoy profiles (ghost mode)
-------------------------------------------------------------------
ghost mode is my favourite feature.

you set a decoy PIN. when someone forces you to unlock — you enter THAT pin instead.

what happens:
→ real session wiped silently
→ decoy profile loads (empty, or pre-configured with fake content)
→ camera/mic vault locked
→ bridge messaging disabled
→ clipboard cleared

they see a clean browser. nothing happened.

TWEET 8 — Camera/mic vault
-------------------------------------------------------------------
camera and mic vault is live.

three modes:
→ Locked — no site can ever request access (hard block at permission handler level)
→ Session — granted until you close the browser
→ Timed — 5 minute window, auto-revokes, you get a countdown

finance container: vault is PERMANENTLY locked. no exceptions. no site can request camera or mic on a finance tab. ever.

-------------------------------------------------------------------
TWEET 9 — Local network encrypted messaging
-------------------------------------------------------------------
built encrypted local mesh messaging. no servers. ever.

how it works:
→ mDNS (bonjour) discovers peers on your LAN
→ WebSocket transport with X25519 ECDH key exchange on connect
→ AES-256-GCM encrypts every message with a per-connection key
→ fresh keypair every launch (perfect forward secrecy)

your public key is broadcast in the mDNS TXT record so peers know who you are before connecting.

no phone numbers. no accounts. just two people on the same wifi.

-------------------------------------------------------------------
TWEET 10 — Argon2id + OS keychain
-------------------------------------------------------------------
upgraded PIN hashing from scrypt to Argon2id.

params:
→ memoryCost: 65,536 (64 MB RAM)
→ timeCost: 3 passes
→ parallelism: 4 threads

brute forcing a 6-digit PIN would take... a while.

also: profile encryption keys are now protected by the OS keychain (safeStorage) — macOS Keychain, Windows DPAPI, Linux libsecret. machine-bound. can't be copied and cracked offline.

TWEET 13 — Password-protected rooms
-------------------------------------------------------------------
rooms in the local mesh can now be password-protected.

creating a room with a password:
→ Argon2id hashes it (same params as PIN: 64MB/3 passes/4 threads)
→ derives a separate 32-byte symmetric key for the room
→ to join: verify the hash, derive the key, you're in

works on top of private rooms (members-only) — you can have both.

all in-memory. wiped on panic. no logs.
